# 财报造假分析系统

专业的财报造假分析系统，用于深度分析上市公司财务报告，识别财务异常和造假风险。

## 项目背景

项目面向上市公司年度报告分析，直接从 PDF 年报中提取财务数据与附注证据，输出风险评分、异常模式、证据摘录和投资建议。

当前重点放在三件事：
- 使用真实年报数据而不是示例数据；
- 让风险结论能回溯到报表原文或附注页码；
- 通过回归测试稳定不同年报版式下的提取结果。

## 核心功能

### 1. 核心业绩背离分析
- 收入与利润背离：检测收入下降但利润增长的异常
- 现金流与利润背离：检测经营现金流与净利润的严重不匹配
- 收入与现金流背离：检测收入与销售现金流的异常关系

### 2. 资产负债表异常检测
- 应收账款异常：检测应收账款异常增长与坏账准备减少
- 存货异常：检测存货积压与跌价准备减少
- 资产质量分析：评估资产真实性和减值风险

### 3. 利润操纵分析
- 非经常性损益分析：检测政府补助等非经常性损益对利润的影响
- 递延收益分析：检测递延收益的突击摊销
- 利润质量评估：评估利润的可持续性和质量

### 4. 运营效率分析
- 产能利用率分析：检测产能利用率下降但仍扩产的异常
- 产销存分析：检测库存积压与销售不畅的矛盾
- 投资效率评估：评估投资决策的合理性

### 5. 公司治理风险分析
- 历史违规记录检查
- 关联交易风险评估
- 内控风险评估

### 6. 造假路径识别
- 收入虚增路径（通过应收账款操纵收入）
- 利润调节路径（通过非经常性损益调节利润）
- 资产虚增路径（少提减值准备虚增资产）
- 现金流操纵路径（利润与现金流严重背离）
- 关联交易路径（通过关联方虚构交易）

### 7. 收入质量分析
- 现金收入比（销售商品收到的现金/营业收入，警戒线80%）
- 应收账款占收入比（警戒线50%）
- 应收账款周转天数异常
- 年末收入集中确认检测

### 8. 费用与成本类造假识别
- 费用资本化异常（研发费用资本化比例过高）
- 折旧政策变更（延长折旧年限减少费用）
- 坏账准备计提比例下降
- 存货跌价准备计提不足

### 9. 资产高估检测
- 在建工程迟迟不转固（利息资本化虚增资产）
- 产能与资产不匹配（产能利用率低但固定资产增长）
- 商誉减值计提不足
- 毛利率与存货周转背离（少结转成本导致存货虚增、毛利率虚高）

### 10. 负债隐瞒检测
- 或有事项披露不足（未决诉讼、对外担保）
- 表外融资嫌疑（SPE/空壳公司转移负债）
- 其他应付款异常

### 11. 报表勾稽校验
- 资产负债表+利润表：未分配利润勾稽关系
- 现金流量表+资产负债表：期末现金变化一致性
- 利润表+现金流量表：经营现金流与净利润匹配性
- 经营现金流/净利润比率（<0.5为极差）

### 12. 现金流质量分析
- 经营现金流/净利润比率评价
- 自由现金流分析（经营现金流-资本支出）
- 现金余额与利息收入匹配性（伪造现金嫌疑）

### 13. 审计与治理分析
- 审计意见解读（标准/保留/否定/无法表示）
- 审计师变更预警
- 审计费用异常波动
- 会计估计变更检测

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt

# 开发模式安装，启用共享 CLI 入口
pip install -e .
```

### 2. 使用命令行工具

```bash
# 分析单个财报文件
python main.py analyze path/to/report.pdf

# 指定公司名称和报告年度
python main.py analyze path/to/report.pdf --company "英洛华科技" --year 2025

# 批量分析多个财报文件
python main.py batch inputs/*.pdf

# 指定输出目录和格式
python main.py analyze path/to/report.pdf --output custom_reports/ --format json

# 显示详细输出
python main.py analyze path/to/report.pdf --verbose

# 使用安装后的共享 CLI 入口
fraud-analyzer path/to/report.pdf
batch-fraud-analyzer inputs/*.pdf --format json
```

### 3. CLI 安装验证

```bash
# 验证共享 CLI 入口与安装说明
python -m pytest tests/test_cli_smoke.py tests/test_project_metadata.py -v
```

### 4. 使用Python API

```python
from src.core.analyzer import FinancialFraudAnalyzer

# 创建分析器
analyzer = FinancialFraudAnalyzer(output_dir="outputs")

# 分析财报
report = analyzer.analyze(
    pdf_path="path/to/report.pdf",
    company_name="英洛华科技",
    report_year=2025
)

# 查看分析结果
print(f"风险等级: {report.risk_assessment.risk_level.value}")
print(f"风险评分: {report.risk_assessment.total_score:.1f}/50")
print(f"投资建议: {report.investment_recommendation}")
```

### 5. 批量分析

```python
from src.core.analyzer import FinancialFraudAnalyzer

analyzer = FinancialFraudAnalyzer(output_dir="outputs")

pdf_files = ["path/to/report1.pdf", "path/to/report2.pdf", "path/to/report3.pdf"]
reports = analyzer.batch_analyze(pdf_files, "outputs")
print(f"成功分析 {len(reports)} 个文件")
```

## 检测指标

### 核心业绩背离指标（7个）
1. 收入与利润背离
2. 现金流与利润背离
3. 收入与现金流背离
4. 扣非净利润与归母净利润背离
5. 政府补助依赖度
6. 非经常性损益占比
7. 递延收益摊销异常

### 资产负债表异常指标（6个）

6. 存货/收入比率

### 运营效率指标（4个）
1. 产能利用率趋势
2. 在建工程异常增长
3. 库存与销售增速差异
4. 生产与销售增速差异

### 公司治理指标（2个）
1. 历史违规记录
2. 关联交易风险

### 收入质量指标（3个）
1. 现金收入比（销售商品收到的现金/营业收入）
2. 应收账款占收入比
3. 应收账款周转天数

### 费用与成本造假指标（4个）
1. 费用资本化比例异常
2. 折旧政策变更
3. 坏账准备计提比例下降
4. 存货跌价准备计提不足

### 资产高估指标（3个）
1. 在建工程迟迟不转固
2. 产能与资产不匹配
3. 商誉减值计提不足

### 负债隐瞒指标（2个）
1. 或有事项披露不足
2. 表外融资嫌疑

### 报表勾稽校验指标（3个）
1. 经营现金流/净利润比率
2. 净利润为正但经营现金流为负
3. 资产-负债=权益校验

### 现金流质量指标（2个）
1. 自由现金流分析
2. 现金余额与利息收入匹配性

### 审计与治理指标（3个）
1. 审计师变更
2. 审计费用异常波动
3. 会计估计变更

### 红警信号（Red Flags）
- 应收账款增速 > 收入增速
- 存货增速 > 成本增速
- 毛利率异常高于行业或异常平滑
- 经营现金流持续为负
- 关联交易占比 > 30%
- 非标审计意见
- 会计政策/估计频繁变更
- 审计师频繁更换
- 毛利率上升 + 存货周转率下降（典型造假特征）

## 风险等级

| 风险等级 | 评分范围 | 说明 |
|---------|---------|------|
| 低风险 | 0-14分 | 财务健康，无明显造假迹象 |
| 中风险 | 14-25分 | 存在一定风险，需要关注 |
| 高风险 | 25-38分 | 存在明显造假迹象，建议谨慎 |
| 极高风险 | 38-50分 | 存在严重造假迹象，建议回避 |

评分结果展示为一行分项累计，例如：

`模式严重度 9.0 + 财务严重度 4.0 + 密度 3.0 + 广度 1.5 + 集中度 2.5 + 最高风险加分 1.0 = 21.0`

分项含义：
- 模式严重度：按风险等级非线性计分（CRITICAL=15, HIGH=8, MEDIUM=3）
- 财务严重度：亏损、收入暴跌、现金流为负等极端情况额外加分
- 风险密度：高风险模式占比（用1.5次方放大差异）
- 风险广度：触发模式数量（越多越危险）
- 风险集中度：高风险指标占比
- 最高风险等级：是否存在CRITICAL级别模式

## 配置说明

### 阈值配置 (configs/thresholds.yaml)
包含盈利能力、偿债能力、运营能力、成长能力、造假检测、风险等级等阈值配置。

### 权重配置 (configs/weights.yaml)
包含造假模式权重、风险指标权重、行业调整权重、公司规模调整权重、时间权重、审计意见权重、历史违规权重、综合风险权重等配置。

### 自定义配置

可通过修改 `configs/thresholds.yaml` 和 `configs/weights.yaml` 调整检测阈值和风险权重，修改后重新运行即可生效。

## 报告格式

系统生成的分析报告包含以下部分：

1. 公司基本信息
2. 财务数据摘要
3. 财务比率分析（盈利能力、偿债能力、运营效率、成长能力）
4. 收入质量分析（现金收入比、应收账款占收入比、周转天数）
5. 现金流质量分析（经营现金流/净利润比率、盈利质量评价）
6. 报表勾稽校验（资产-负债=权益等）
7. 造假检测结果
8. 风险评估
9. 投资建议
10. 详细分析
11. 建议措施

报告支持 Markdown 和 JSON 两种输出格式。
